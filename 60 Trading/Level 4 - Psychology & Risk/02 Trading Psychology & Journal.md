# 02 Trading Psychology & Journal

## 🔹 ความจริงที่เทรดเดอร์ใหม่ไม่ยอมรับ

**90% ของเทรดเดอร์เจ๊ง — ไม่ใช่เพราะวิเคราะห์ไม่เป็น แต่เพราะ Psychology พัง**

คนส่วนใหญ่เข้าใจผิดว่า:
- "ถ้าวิเคราะห์เก่ง = เทรดรอด" → **ผิด**
- "ยิ่งรู้ indicator เยอะ = ยิ่งชนะ" → **ผิด**
- "แพ้เพราะดวง / เพราะตลาด" → **ผิด**

**ความจริง:**
- การเทรดไม่ใช่ศาสตร์ของ "ถูกหรือผิด" — แต่มันคือ **การบริหารอารมณ์ในความไม่แน่นอน**
- เงินจริงทำให้คนเปลี่ยน — เห็น lot ใหญ่ → สมองส่วนสัญชาตญาณ (Amygdala) ทำงาน → ตัดสินใจแย่
- Strategy ที่ดีที่สุดก็พังได้ถ้าจิตใจพัง

### ปัญหาที่พบบ่อย:

| ปัญหา | % เทรดเดอร์ที่เจอ | สาเหตุ | รอดเพราะ |
|-------|-----------------|--------|----------|
| **FOMO** | 90% | กลัวพลาดโอกาส | มี Checklist |
| **Revenge Trading** | 70% | โมโหตอนเสีย | 2-Day Loss Rule |
| **Overtrading** | 80% | ติดเทรด (addiction) | Quality > Quantity |
| **ไม่ตั้ง SL** | 60% | หวังว่าตลาดจะกลับ | Concept 2 SL Placement |
| **ขยับ SL** | 50% | ไม่อยากขาดทุน | Discipline |
| **ปิด TP เร็ว** | 80% | กลัวกำไรหาย | TP = กฏ, ห้ามแตะ |

## 🔹 FOMO — Fear Of Missing Out

### FOMO คืออะไร? ทำไมถึงเกิด?

**FOMO = กลัวพลาดโอกาส — เป็นสัญชาตญาณของมนุษย์ที่ฝังมาแต่โบราณ**

สมองมนุษย์ถูกออกแบบมาให้ "กลัวพลาด" เพราะในยุคหิน — พลาดอาหาร = ตาย  
แต่ในตลาดการเทรด — "พลาด" ไม่ได้แปลว่าตาย

```
อาการ FOMO:
-------------------------------------------------------------------------
"ราคาวิ่งไปแล้ว! ยังไม่เข้าเลย! — จะเข้าทีหลังได้ไหม?"
"OB นั้นเด้งไปแล้วเมื่อวาน — เสียดาย!"
"ทุกคนบอกว่าขึ้น — ฉันก็ต้องเข้าบ้าง!"
-------------------------------------------------------------------------
```

**สิ่งที่ FOMO ทำให้คุณทำ:**

```
1. Chase (ตามราคา):
   - OB ที่ 4500 ← ราคามา 4520 แล้ว
   - FOMO กระซิบ: "เดี๋ยวมันก็ไป 4550 — รีบเข้า"
   - Reality: ถ้ามันจะไป 4550 — รอ pullback เทส 4500 ก่อนก็ได้
   - ผล: ซื้อของแพงกว่าที่ควร → SL กว้าง → risk สูง
   
2. ข้าม confluence:
   - ปกติต้องมี OB + FVG + MTF match → FOMO → แค่ OB ก็พอ
   - ผล: trade คุณภาพต่ำ → ชนะน้อยลง
   
3. Lot ใหญ่กว่าปกติ:
   - "พลาดรอบที่แล้ว — รอบนี้ต้องได้เยอะ"
   - ผล: เสียแล้วเจ็บหนัก → จิตใจพัง
   
4. ไม่ตั้ง SL / ขยับ SL:
   - "เดี๋ยวมันก็กลับ — แค่รอหน่อย"
   - ผล: loss ใหญ่ → revenge trade → cycle
```

### วิธีแก้ FOMO — ใช้ตรรกะสู้สัญชาตญาณ

**1. Pre-Trade Checklist — ใช้ทุกครั้งไม่เว้น**

```
☐ HTF direction match?
☐ OB/FVG มีจริง?
☐ Entry มีเหตุผลจาก market structure?
☐ RR ≥ 1:2?
☐ Confluence ≥ 2?
```

ถ้าตอบ checklist ไม่ผ่าน 1 ข้อ → **ไม่เข้า**  
FOMO จะบอกว่า "ไม่เป็นไร แค่ครั้งเดียว" — อย่าฟังมัน

**2. เข้าใจ: "เสียดาย" ≠ "เสียเงิน"

```
เสียดายที่ไม่ได้เข้า OB = ไม่เสียเงินจริง — แค่เสียดาย
เสียดายที่เข้าแล้วเจอ SL = เสียเงินจริง — ต้องกู้คืน
```

จำไว้ว่า: **ตลาดเปิดทุกวัน** — โอกาสมีไม่จำกัด  
เงินที่คุณมี = จำกัด  
→ อย่าใช้เงินจำกัดเพื่อตามหาโอกาสไม่จำกัด

**3. ตั้งกฏเรื่อง Chase**

```
พอราคาวิ่งเลย OB ไป 10 pip → ห้ามเข้า trade นั้นเด็ดขาด
→ รอรอบหน้า — มีตลาดทุกวัน
```

## 🔹 Revenge Trading — แก้แค้นตลาด

### Revenge Trading คืออะไร?

**Revenge Trading = เสียแล้วอยากเอาคืน — เข้า trade ห่วยๆ เพื่อ "เอาคืน"**

เป็นสัญชาตญาณเดียวกับ "เอาคืนเพื่อนที่แย่งของ"  
แต่ตลาดไม่มีความรู้สึก — คุณจะเอาคืนไม่ได้ — มีแต่เสียเพิ่ม

### วงจร Revenge Trading — รู้แล้วหยุดให้ทัน

```
                 ╔══════════════════╗
                 ║  เสีย Trade #1  ║  ← ปกติ — loss 2% ตามแผน
                 ╚══════════════════╝
                         ↓
             "กูไม่ยอม! ต้องเอาคืน!"
                         ↓
    ╔═══════════════════════════════════╗
    ║  เทรดโดยไม่รอ confluence         ║
    ║  Lot ใหญ่ขึ้น (×1.5-2)            ║
    ║  SL กว้างขึ้น / ไม่ตั้ง SL         ║
    ╚═══════════════════════════════════╝
                         ↓
                 ╔══════════════════╗
                 ║  เสีย Trade #2  ║  ← หนักขึ้น — loss 4-5%
                 ╚══════════════════╝
                         ↓
             "กูต้องเอาคืนให้ได้!"
                         ↓
    ╔═══════════════════════════════════╗
    ║  Lot ×3                          ║
    ║  ไม่สน TF ไม่สน S/R               ║
    ║  "เข้าเดา — ไม่เข้า analysis"     ║
    ╚═══════════════════════════════════╝
                         ↓
                 ╔══════════════════╗
                 ║  เสีย Trade #3  ║  ← หนักมาก — loss 10-15%
                 ╚══════════════════╝
                         ↓
              ╔════════════════════════╗
              ║  ทุนหาย 20-50%        ║
              ║  → ถอดใจ / ฝากเพิ่ม   ║  ← จุดจบของrevenge cycle
              ╚════════════════════════╝
```

### ทำไม Revenge Trading ถึงไม่ work — คณิตศาสตร์

```
สมมติ: ปกติ Win Rate 50%, RR 1:2
  Trade 1: เสีย $20 (2%)
    → ตอนนี้ติดลบ $20
    → ต้องเทรด 1 ครั้งปกติเพื่อเอาคืน

  Trade 2 (revenge): Lot ×2 → เสี่ยง $40
    → เสียอีก: ตอนนี้ติดลบ $60
    → ต้องเทรด 3 ครั้งปกติเพื่อเอาคืน

  Trade 3 (revenge ×3): เสี่ยง $60
    → เสียอีก: ตอนนี้ติดลบ $120 (12% ของทุน)
    → ต้องเทรด 6 ครั้งปกติเพื่อเอาคืน
```

**การ revenge = เอาเลขคณิตมาฆ่าตัวเอง** — ยิ่งทำยิ่งต้องใช้ trades มากขึ้นเพื่อเอาคืน

### วิธีแก้ — 2-Day Loss Rule

```
กฏเหล็ก: ถ้าเสีย 2 ครั้งติด → หยุดเทรดวันนั้นทันที

"หยุด" = 
  ✅ ปิด MT5
  ✅ ปิดกราฟ
  ✅ ไปเดินเล่น / ดูหนัง / อ่านหนังสือ
  ✅ กลับมาพรุ่งนี้ด้วยสมองที่เย็น

"ห้าม" =
  ❌ เปิด MT5 อีก "แค่ดูเฉยๆ"
  ❌ เปิดบัญชีเดโม่เทรดต่อ
  ❌ กลับมาวิเคราะห์ว่า "ทำไมเสีย"
```

### สัญญาณเตือน — เช็คก่อนเข้า trade ถัดไป

```
☐ เปิดกราฟแล้วรู้สึก "โกรธ" หรือ "คันไม้คันมือ"?
☐ อยากเทรดเพื่อ "เอาคืน"?
☐ Lot ใหญ่กว่าที่คำนวณไว้?
☐ ข้าม confluence / checklist?
☐ เข้าเพราะ "รู้สึก" ไม่ใช่ "วิเคราะห์"?

→ ถ้าใช่ข้อใดข้อหนึ่ง → หยุด!
```

## 🔹 Overtrading — เทรดมากเกินไป

### Overtrading คืออะไร?

**Overtrading = เทรดมากกว่าที่ควร — ทั้งๆ ไม่มีโอกาสคุณภาพ**

FOMO กับ Overtrading ต่างกัน:
- FOMO = กลัวพลาด opportunity → เข้า trade ที่ไม่ดี
- Overtrading = **ติดเทรด** → ต้องเข้า trade ตลอดเวลา → quality ↓

### ทำไม Overtrading ถึงอันตราย?

```
1. Spread กินกำไร
   - เปิด 10 trades ต่อวัน = เสีย spread 10 ครั้ง
   - สมมติ spread 3 pips = เสีย 30 pip ต่อวัน
   - กำไรทั้งวัน 50 pip → หัก spread 30 pip → เหลือ 20 pip
   - เทรด 2 ครั้ง → ได้ 50 pip → หัก spread 6 pip → เหลือ 44 pip
   
2. Analysis Fatigue (ล้า)
   - ยิ่งเทรดเยอะ → สมองล้า → ตัดสินใจแย่ลง
   - Trade #1: วิเคราะห์ดี → ชนะ
   - Trade #3: เหนื่อย → ข้าม confluence → เสีย
   - Trade #5: ล้ามาก → revenge trade → หนัก
   
3. Law of Diminishing Returns
   - 1 trade ที่ดี: RR 1:3 + win = +3 units
   - 5 trade ที่ห่วย: RR 1:1, win rate 40% = (+1+1+1-1-1) = +1 unit
   - ใช้เวลามากกว่าแต่ได้น้อยกว่า
   
4. Win Rate ตก
   - trade ห่วยเยอะ → ชนะน้อย → confidence ตก → quality ห่วยไปอีก
```

### อาการ Overtrading — ดูว่าตัวเองเป็นไหม

```
☐ "วันนี้ยังไม่ได้เทรดเลย" → รู้สึกว่าต้องหา trade
☐ เปิด 5+ trades ใน 1 ชม
☐ เข้า trade ที่ confluence < 2
☐ เปิดกราฟแล้วขยับ Entry ตลอดเวลา
☐ รู้สึกกระสับกระส่ายตอนไม่เทรด
☐ เทรดในเวลาที่ไม่ควร (ข่าว, midnight, etc.)
```

### วิธีแก้

```
✅ Quality > Quantity
   - 1 trade คุณภาพดี = 10 trade คุณภาพห่วย
   - เปรียบ: อาหาร Michelin star จานเดียว > ข้าวกล่อง 10 กล่อง

✅ Limits ต่อวัน
   - มือใหม่: สูงสุด 2 trades/วัน
   - มีประสบการณ์: สูงสุด 3-5 trades/วัน
   - ไม่มี limit = ไม่มีวินัย

✅ Confluence ≥ 2 เสมอ
   - OB + FVG = OK
   - OB อย่างเดียว = ไม่พอ → ข้าม
   - OB + FVG + MTF match = ดี

✅ จดจำ: "ไม่เข้า = ชนะ"
   - ใน Concept 3: "Win by not trading"
   - การข้าม trade ที่ไม่แน่ใจ = เก็บเงินไว้สำหรับ trade ที่แน่
```

## 🔹 Premature Exit — ปิดเร็วเกินไป

### ปัญหา

```
Scenario:
  1. วิเคราะห์: OB Buy ที่ 4500, SL 4497, TP 4520
  2. เปิด Buy ที่ 4500 ✅
  3. ราคาวิ่งไป 4510
  4. "โอ้ย ได้ 10 จุดแล้ว! ปิดดีกว่า กลัวมันกลับ"
  5. ปิด → ได้ 10 จุด
  6. 2 ชมต่อมา → ราคาไป 4520 (TP)
  7. "เฮ้ย! ถ้าไม่ปิดก็ได้อีก!"
```

**Premature Exit = ขโมยกำไรของตัวเอง — ระบบพังเพราะตัวเอง**

### ทำไมคนถึงปิดเร็ว? — เข้าใจสาเหตุ

```
1. กลัวกำไรหายไป
   → สมองประเมิน: "10 จุดตอนนี้ ดีกว่า 20 จุดในอนาคต"
   → แต่มันคือ fallacy — เพราะระบบมี edge
   
2. ขาดความมั่นใจในระบบ
   → "OB จริงป่าวนะ...?"
   → ถ้าไม่มั่นใจ → ไม่ควรเข้าแต่แรก
   → ถ้าเข้าแล้ว → ให้ระบบทำงาน
   
3. ไม่มี TP ตั้งแต่แรก (หรือตั้งไว้แต่ไม่เชื่อ)
   → เข้าแบบ "เดา" → ไม่รู้ว่ารอถึงไหน
   → พอได้นิดหน่อย → รีบปิด

4. กลัว MSS (Market Structure Shift)
   → "แล้วถ้ามันกลับล่ะ..."
   → จุดนี้ต้อง trust the system
```

### วิธีแก้ — ใช้ระบบสู้

```
✅ ตั้ง TP ก่อนเข้า — แล้วห้ามแตะ
   - TP = OB/FVG ถัดไป (มีเหตุผล)
   - "TP = กฏ" → กฏห้ามแหก

✅ ใช้ Trailing Stop แทนการปิดเอง
   - ระบบปิดให้ → ไม่ต้องใช้อารมณ์ตัดสินใจ
   - ตั้ง break-even เมื่อราคาไปถึง TP × 0.5

✅ ศึกษา Concept 2 — SL TP Placement
   - TP ที่ดี = level ที่ตลาดเคยตอบสนอง
   - ถ้าเชื่อว่าระดับนั้นถูก → ปล่อยให้มันทำงาน

✅ ฝึก — "Let the trade play out"
   - 10 trades ที่ปล่อยให้ TP ทำงาน
   - vs 10 trades ที่ปิดก่อน
   - บันทึกว่า แบบไหนได้มากกว่ากัน
```

## 🔹 Analysis Paralysis — วิเคราะห์มากเกินไป

### Analysis Paralysis คืออะไร?

**Analysis Paralysis = วิเคราะห์มากเกินไปจนไม่กล้าเข้า trade**

เป็นปัญหาที่ตรงข้ามกับ FOMO/Overtrading:  
- FOMO = เข้าแบบไม่คิด  
- Analysis Paralysis = คิดมากเกินไปจนไม่เขาสักที

### อาการ

```
"RSI บอก Buy, MACD บอก Sell, Stochastic กำลังจะตัด... แล้วยังไง?"
"OB ตรงนี้ — แต่ FVG ไม่ซ้อนนะ — รออีก OB ไหม"
"รอดูอีก 1 candle — รอดูอีก 1 — ราคาวิ่งไปแล้ว"
"ถ้าเข้า Buy → แล้วเกิด news ขึ้น → แล้ว SL โดน → แล้ว..."
```

### สาเหตุ

```
1. กลัวผิด — "ถ้าวิเคราะห์ไม่ครบ = ผิด = เสียเงิน"
   → ความจริง: ไม่มี анализы ไหน perfect — ต้องยอมรับความไม่แน่นอน

2. ใช้ indicator มากเกินไป
   → ยิ่งมีตัวแปรเยอะ → ยิ่งขัดแย้งกันเอง → ไม่ได้ข้อสรุป

3. ไม่มี framework ตายตัว
   → เปิดกราฟ → ดูนั่นดูนี่ → ไม่รู้จะเริ่มตรงไหน
```

### วิธีแก้

```
✅ ใช้ Framework เดียวกันทุกครั้ง
   HTF → Trend → OB → FVG → Entry → SL → TP
   ทำเป็นขั้นตอน — ไม่ข้าม — ไม่เพิ่ม

✅ Indicator สูงสุด 2 ตัว
   RSI (momentum) + 1 indicator เสริมพอ
   indicator 3+ = noise

✅ Timebox — 5 นาที
   วิเคราะห์ 5 นาที → ถ้ายังไม่ชัด → ข้าม
   5 นาทีพอสำหรับการวิเคราะห์ที่มีคุณภาพ
   ถ้าต้องใช้ 30 นาที = ยังไม่ชัดพอ = ข้าม

✅ จำ: "ดีกว่าพลาด = เทรดห่วย"
   เสียดายที่ไม่ได้เข้า = เสียดาย
   เสียหายที่เข้าไม่ดี = เสียเงิน
```

## 🔹 Trading Journal — ทำไมสำคัญที่สุดในบรรดาทั้งหมด

### Journal ไม่ใช่แค่ "จด" — มันคือเครื่องมือพัฒนาตัวเอง

คน 90% ไม่จด journal → ทำผิดซ้ำๆ  
คน 10% ที่จด → เห็น pattern → ปรับปรุง → รอด

### Journal ช่วยอะไรบ้าง?

```
1. หา Pattern — "ผมเสียเพราะอะไรบ่อย?"
   → อาจเป็นเพราะ: FOMO ตอน 3 ทุ่ม, Revenge หลังเสีย 2 ครั้ง
   → พอรู้ pattern → ป้องกันได้

2. วัดผลระบบ — "Win rate จริงเท่าไหร่?"
   → คนส่วนใหญ่จำว่า "เทรดดี" → journal บอกความจริง
   → "Win rate 30%? งั้นต้องปรับระบบ"

3. หาจุดอ่อน — "ทำไม Trade กลางคืนเสีย?"
   → เพราะเวลา 3 ทุ่ม = ตลาดเอเชียเปิด → low volume → noise
   → วิธีแก้: ไม่เทรด 3 ทุ่ม

4. ดู Psychology — "ทำไม trade นี้ถึงเสีย?"
   → อาจเป็นเพราะ: "โมโหจาก trade ก่อน" ไม่ใช่เพราะ OB ไม่ work
```

### สถิติที่ต้อง Track รายสัปดาห์

```
1. Total Trades — จำนวน trades ทั้งหมด
2. Win Rate (%) — % trades ที่ชนะ
3. Avg Win — กำไรเฉลี่ยต่อ trade ที่ชนะ ($)
4. Avg Loss — ขาดทุนเฉลี่ยต่อ trade ที่เสีย ($)
5. Avg RR (Win) — RR เฉลี่ยของ trades ที่ชนะ
6. Avg RR (Loss) — RR เฉลี่ยของ trades ที่เสีย
7. Profit Factor = Win Total / Loss Total
   → ≥ 2 = ดี, ≥ 1 = OK, < 1 = พัง
8. Max Consecutive Losses — เสียติดกี่ครั้ง
   → ถ้า > 5 → มีปัญหา
9. Max Drawdown — จุดต่ำสุดจาก peak
10. Expectancy = (Win Rate × Avg Win) − (Loss Rate × Avg Loss)
```

### ตัวอย่างการวิเคราะห์ Journal จริง

```
สมมติ journal 1 เดือน:
  Total Trades: 60
  Win Rate: 45%
  Avg Win: $25 (RR 1:2.5)
  Avg Loss: $12 (RR 1:1.2)
  Profit Factor: (27 × $25) / (33 × $12) = $675 / $396 = 1.70 ✅
  Max Consecutive Losses: 4 ครั้ง
  Max Drawdown: 8%

วิเคราะห์:
  - ระบบ profitable (PF 1.70)
  - แต่ max losses 4 ครั้ง = จิตใจต้องรับไหว
  - ถ้า Max DD 8% → risk 2% ต่อ trade → 4 loss = 8% → ตรงกับ max DD
  - → ใช้ 2% ต่อ trade = OK
  
ปรับปรุง:
  - Avg Loss $12 (RR 1:1.2) = เสียเยอะกว่าที่ควร
  - → review SL placement → ปรับให้ SL แคบลง
  - → อาจใช้แม่นขึ้น → loss น้อยลง → PF สูงขึ้น
```

## 🔹 Concept 3 — วินัยที่จำง่าย

> **"Win by not trading — อยู่รอดก่อนรวย"**

3 ข้อที่ต้องจำ:

1. **ไม่เข้า = ชนะ** — ข้าม trade ห่วย = ได้เงิน
2. **ตลาดไม่ไปไหน — เงินคุณไปไหนไม่ได้** — เงินสดคือพลัง
3. **เทรดเดอร์ที่ดี ≠ ถูกทุกครั้ง — เทรดเดอร์ที่ดี = รอดทุกครั้ง**
