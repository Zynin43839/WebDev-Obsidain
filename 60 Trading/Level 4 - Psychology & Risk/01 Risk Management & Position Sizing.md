# 01 Risk Management & Position Sizing

## 🔹 ทำไม Risk Management ถึงสำคัญที่สุด

**ถ้าไม่มี Risk Management — ระบบวิเคราะห์ที่ดีที่สุดก็พัง**

คนส่วนใหญ่คิดว่าการเทรด = หาจุดเข้าที่แม่นยำ  
แต่ความจริง: **การเทรด = การบริหารความน่าจะเป็น (Probability Management)**

ต่อให้ระบบวิเคราะห์แม่น 70% — ถ้าเสียทีละ 10% → 10 ครั้งติด (ซึ่งมีโอกาสเกิด 0.3¹⁰ ≈ 0.0006% แต่ในชีวิตจริงมันเกิดได้) → เงินหาย 65%

```
ตัวอย่าง:
  ทุน $1,000
  เสีย 10 ครั้งติด ครั้งละ 10% ($100)
  → เงินคงเหลือ: $1,000 → $900 → $810 → $729 → $656 → $590
                  → $531 → $477 → $430 → $387 → $348
  → ต้องกำไร 187% ถึงกลับมาที่ทุนเดิม!
  → 187% = เทรดถูก 18.7 ครั้งติด ครั้งละ 10% — แทบเป็นไปไม่ได้

  vs
  เสีย 10 ครั้งติด ครั้งละ 2% ($20)
  → เงินคงเหลือ: $1,000 → $980 → $960 → $941 → $922 → $904
                  → $886 → $868 → $851 → $834 → $817
  → ต้องกำไรแค่ 22% ถึงกลับมาที่ทุนเดิม
```

**ข้อแตกต่าง:** เสีย 2% → ฟื้นตัวง่าย | เสีย 10% → เกือบตาย

### ทำไมคนถึง ignore Risk Management?

1. **เข้าใจผิดว่าตัวเองไม่เสียติดกันหลายครั้ง** — ความจริง: streak เกิดขึ้นเสมอ
2. **มั่นใจเกิน (Overconfidence)** — ระบบเทพ → risk น้อยไปทำไม
3. **ชอบ adrenaline** — เห็น lot ใหญ่แล้วตื่นเต้น
4. **ไม่เข้าใจ concept of expectancy** — โฟกัสที่ถูก/ผิด มากกว่า expectancy รวม

### สิ่งที่ RM ปกป้อง:
1. **ทุน (Capital Preservation)** — พื้นฐานของทุกอย่าง
2. **จิตใจ (Psychology)** — เสียทีละน้อย → ใจเย็น → เทรดดี
3. **โอกาส (Opportunity)** — มีเงินเหลือ = มีโอกาสเทรดวันหน้า
4. **Compound Effect** — ค่อยๆ กำไร = โตเร็วในระยะยาว

## 🔹 กฏ 1-2% — หัวใจของ Risk Management

### กฏ

```
เสียไม่เกิน 1-2% ของทุนต่อ 1 trade

ทุน $1,000 → เสี่ยง $10-20 ต่อ trade
ทุน $10,000 → เสี่ยง $100-200 ต่อ trade
```

### ทำไม 1-2%?

```
เสีย 2% 10 ครั้งติด → เหลือ 81.7% → ต้องกลับมา 22.3%
เสีย 5% 10 ครั้งติด → เหลือ 59.9% → ต้องกลับมา 67.0% ← เริ่มยาก
เสีย 10% 10 ครั้งติด → เหลือ 34.9% → ต้องกลับมา 187% ← แทบเป็นไปไม่ได้
```

### ข้อยกเว้น

| ทุน | Max Risk/Trade | หมายเหตุ |
|-----|---------------|----------|
| $100-500 | 2% | ทุนน้อย — risk สูงหน่อยเพื่อให้ trade ได้ |
| $500-5,000 | 1.5% | ปานกลาง |
| $5,000+ | 1% | ทุนมาก — 1% ก็พอ |

## 🔹 Position Sizing — คำนวณ Lot Size ให้ตรงกับ Risk

### ทำไมต้องคำนวณ Lot Size? — ไม่ใช่แค่ "กะๆ"

หลายคนคิดว่า "ทุน $1,000 → เปิด 0.10 lot ก็ปลอดภัยแล้ว"  
แต่ความจริง: **0.10 lot XAUUSD = $1/pip** → ถ้า SL กว้าง 30 pip → เสี่ยง $30 = 3% ของทุน → เกิน 2%

**Lot Size ไม่ได้ตายตัว — มันขึ้นอยู่กับ 3 อย่าง:**
1. **Account Balance** — มีเงินเท่าไหร่
2. **Risk %** — ยอมเสียกี่ % ต่อ trade
3. **SL Distance** — SL กว้างเท่าไหร่ (ขึ้นกับ market structure)

### สูตร — จำให้ขึ้นใจ

```
Lot Size = (Account Balance × Risk %) ÷ (SL Distance × Pip Value)
```

**อธิบายแต่ละส่วน:**

| ส่วน | ตัวอย่าง | ความหมาย |
|------|----------|----------|
| **Account Balance** | $1,000 | เงินทั้งหมดที่มี |
| **Risk %** | 2% = 0.02 | เปอร์เซ็นต์ที่ยอมเสีย |
| **Account × Risk%** | $1,000 × 0.02 = $20 | เงินที่ยอมเสียใน trade นี้ |
| **SL Distance (pip)** | 10 pip | ระยะจากเข้า→SL |
| **Pip Value** | $0.10 | 1 pip = เท่าไหร่ |
| **SL Dist × Pip Val** | 10 × $0.10 = $1 | เสียเงินต่อการลด 1 lot |
| **สูตรเต็ม** | $20 ÷ $1 = 0.20 lots | Lot ที่เปิดได้ |

### ตัวอย่าง Scenario จริง:

```
Scenario 1: SL แคบ
  ทุน $1,000, Risk 2% ($20)
  SL = 5 pip
  Pip Value = $0.10
  Lot = $20 ÷ (5 × $0.10) = $20 ÷ $0.50 = 0.40 lot
  → เสี่ยง $20 เท่ากัน แต่ lot ใหญ่เพราะ SL แคบ

Scenario 2: SL กว้าง
  ทุน $1,000, Risk 2% ($20)
  SL = 50 pip
  Pip Value = $0.10
  Lot = $20 ÷ (50 × $0.10) = $20 ÷ $5 = 0.04 lot
  → Lot เล็กเพราะ SL กว้าง — ต้องลด lot เพื่อให้ risk เท่าเดิม
```

> **หัวใจ:** ไม่ว่า SL จะแคบหรือกว้าง — **เสี่ยงเงินเท่าเดิมเสมอ**  
> SL แคบ = lot ใหญ่, SL กว้าง = lot เล็ก

### Pip Value สำหรับ XAUUSD

| Type | Pip | Pip Value (1 lot) |
|------|-----|-------------------|
| GOLD (XAUUSD) ปกติ | 0.10 | $10 |
| GOLDmicro | 0.10 | $0.10 |
| GOLDmini | 0.10 | $1 |

**สังเกต:**  
XAUUSD 1 lot = $10/pip → ถ้าทุน $1,000 → 1 pip = 1% ของทุน → แพงมาก  
→ มือใหม่ส่วนใหญ่ใช้ GOLDmicro ($0.10/pip) เพื่อบริหารความเสี่ยง

### ตาราง Lot Size สำเร็จรูป

**ทุน $1,000 — Risk 2% ($20):**

| SL Distance | Pip Value | Lot Size | เสี่ยง (ระยะส้ัน) |
|-------------|-----------|----------|----------------|
| 5 pip | $0.10 | 0.40 | 5 × $0.10 × 0.40 = $20 ✅ |
| 10 pip | $0.10 | 0.20 | 10 × $0.10 × 0.20 = $20 ✅ |
| 20 pip | $0.10 | 0.10 | 20 × $0.10 × 0.10 = $20 ✅ |
| 50 pip | $0.10 | 0.04 | 50 × $0.10 × 0.04 = $20 ✅ |

**ทุน $500 — Risk 2% ($10):**

| SL Distance | Pip Value | Lot Size | เสี่ยง |
|-------------|-----------|----------|-------|
| 5 pip | $0.10 | 0.20 | $10 ✅ |
| 10 pip | $0.10 | 0.10 | $10 ✅ |
| 20 pip | $0.10 | 0.05 | $10 ✅ |
| 50 pip | $0.10 | 0.02 | $10 ✅ |

**ข้อควรจำ:**
- ตารางนี้ใช้สำหรับ GOLDmicro ($0.10/pip) เท่านั้น
- ถ้าใช้ GOLDmini ($1/pip) → ต้องหาร lot size ด้วย 10
- ถ้าใช้ GOLD ปกติ ($10/pip) → ต้องหาร lot size ด้วย 100

## 🔹 Drawdown Management

### Drawdown คืออะไร? — ตัววัดสุขภาพบัญชี

**Drawdown = % ที่บัญชีลดลงจากจุดสูงสุด**  
ไม่ใช่ดูแค่วันนี้เทียบเมื่อวาน — ดูจาก **peak (จุดสูงสุดที่บัญชีเคยไปถึง)**

```
ตัวอย่าง:
  เดือน 1: ทุน $1,000 → $1,200 (peak = 1,200)
  เดือน 2: $1,200 → $900  = Drawdown 25% จาก peak
  → สมองคุณจะคิดว่า "เสียแค่ $300 เอง"
  → แต่จริงๆ = 25% จาก peak — ต้องกำไร 33% ถึงกลับมา peak
```

**ทำไม Drawdown อันตรายกว่า % Loss ต่อ trade?**

| Drawdown | ต้องกลับมากำไรเท่าไหร่ | ความสามารถ |
|----------|----------------------|-----------|
| 10% | 11.1% | ปกติ — ฟื้นไว |
| 20% | 25.0% | เริ่มยาก |
| 30% | 42.9% | ยากมาก |
| 40% | 66.7% | เกือบตาย |
| 50% | 100% | ตาย — ต้องเทรดให้ได้ 100% ถึงเท่าทุน |

**ความสัมพันธ์: Drawdown ยิ่งมาก → % ที่ต้องกลับมา = ยิ่งมากกว่า % ที่เสียไป**  
เพราะฐานทุนเล็กลง — ต้องกำไรจากฐานที่เล็กลงให้ได้ % เท่าเดิม

### กฏ Drawdown — Safety Level

```
Drawdown 10% → ลด lot size เหลือครึ่ง
  → แปลว่า: ระบบกำลังมีปัญหา (ไม่ว่าเหตุผลอะไร)
  → อาจเป็น: ตลาดเปลี่ยน, จิตใจพัง, system ไม่ work
  → ลดความเสี่ยงก่อน — อย่ารอให้หนักขึ้น

Drawdown 15% → หยุดเทรด 1 สัปดาห์
  → หยุด = หยุดจริง — ห้ามเปิด MT5
  → ใช้วันนั้น review journal, หาจุดบอด
  → ตอบคำถาม: "อะไรที่ทำผิดตั้งแต่แรก?"

Drawdown 20% → หยุดเทรด 1 เดือน
  → 20% = systemic failure — ไม่ใช่แค่โชคร้าย
  → กลับไปเรียนใหม่: paper trade จนกว่าจะ stable
  → ถึงจะกลับมา — ใช้ lot เล็กกว่าเดิม 50%
```

### สาเหตุที่ Drawdown เกิดขึ้น (และวิธีป้องกัน)

| สาเหตุ | เกิดกับใคร | วิธีแก้ |
|--------|-----------|--------|
| Market Regime Change | ทุกคน | รอสังเกต — เปลี่ยน strategy |
| Psychology พัง | 70% ของเทรดเดอร์ | หยุดทันที, journal |
| Overleverage | 80% | ลด lot size |
| Risk % มากเกินไป | 60% | ใช้ 1-2% เท่านั้น |
| No system | 50% | กลับไปเรียน |

### Equity Curve — รูปพรรณของเทรดเดอร์

**Equity Curve = กราฟเส้นเงินในบัญชีตลอดเวลา**

```
เส้นที่ดี:
  ↗↗↗↘↗↗↗↘↗↗↗↗↘
  - ค่อยๆ ขึ้น (เทรดเดอร์ที่ profitable)
  - ขาดทุนสั้น (1-2 ครั้งติด)
  - ลงแล้วเด้งกลับสูงกว่าเดิม

เส้นที่แย่:
  ↗↘↗↘↘↗↘↘↘↗↘↘
  - ไม่ไปไหน (เทรดเดอร์ breakeven)
  - ขาดทุนและกำไรสลับ
  - ต้องกลับไปปรับ system

เส้นที่พัง:
  ↘↘↘↗↘↗↗↘↘↘↘↘
  - ลงต่อเนื่อง (เทรดเดอร์ที่เจ๊ง)
  - มีดีขึ้นบ้างแต่ก็ลงหนักกว่า
  - = การพนัน ไม่ใช่การเทรด
```

**วิธีใช้ Equity Curve:**
- ถ้า equity curve เกิน 3 เดือนแล้วยัง flat → system มีปัญหา → ปรับหรือเปลี่ยน
- ถ้าลง 3 เดือนติด → หยุดจริง — กลับไปเรียน
- ถ้าขึ้น 3 เดือนติด → system ใช้ได้ — อย่าเปลี่ยน

## 🔹 RR (Risk:Reward Ratio)

### ความหมาย

```
RR = TP Distance ÷ SL Distance

SL = 10 pip → TP = 20 pip → RR = 1:2
SL = 10 pip → TP = 30 pip → RR = 1:3
SL = 10 pip → TP = 10 pip → RR = 1:1
```

### Win Rate vs RR — ของ 2 อย่างที่ต้องบาลานซ์

คำถาม: "Win Rate 70% ดีกว่า Win Rate 40% ไหม?"

ตอบ: **ไม่เสมอไป — ขึ้นกับ RR**

```
ตัวอย่าง Realistic:
  เทรดเดอร์ A: Win Rate 70%, Avg RR 1:0.5
    → 70 win × 0.5 = +35 units
    → 30 loss × 1 = -30 units
    → Total = +5 units ต่อ 100 trades (แทบไม่กำไร)

  เทรดเดอร์ B: Win Rate 40%, Avg RR 1:3
    → 40 win × 3 = +120 units  
    → 60 loss × 1 = -60 units
    → Total = +60 units ต่อ 100 trades (กำไรดี)
```

**สรุป:**  
- Win Rate สูง ≠ ดีกว่า ถ้า RR ต่ำ  
- Win Rate ต่ำ ≠ แย่กว่า ถ้า RR สูงพอ  
- สิ่งที่สำคัญ = **Expectancy = (Win Rate × Avg Win) − (Loss Rate × Avg Loss)**

### Expectancy — วิธีบอกว่าระบบนี้รอดไหม

```
Expectancy = (Win Rate × Avg Win) − (Loss Rate × Avg Loss)

ตัวอย่าง:
  Win Rate = 40%, Avg Win = 3 units, Avg Loss = 1 unit
  Expectancy = (0.40 × 3) − (0.60 × 1) = 1.20 − 0.60 = +0.60

  → ทุก $1 ที่เสี่ยง → ได้กลับมา $0.60
  → ระบบ +EV (profitable)
```

| Expectancy | แปลว่า | ควรทำ |
|------------|--------|-------|
| **+** | ระบบ profitable | scale ได้ |
| **0** | breakeven | ปรับ |
| **-** | ระบบไม่รอด | เปลี่ยน |

### ตาราง Win Rate × RR ที่ profitable

| Win Rate | Min RR เพื่อ profitable | ตัวอย่าง |
|----------|------------------------|----------|
| 30% | ≥ 1:2.33 | 1:3 → OK |
| 40% | ≥ 1:1.50 | 1:2 → OK |
| 50% | ≥ 1:1.00 | 1:1 → OK |
| 60% | ≥ 1:0.67 | 1:1 → OK |
| 70% | ≥ 1:0.43 | 1:1 → OK |

> **แนวทาง SMC ที่แนะนำ:** RR ≥ 1:2 + Win Rate ≥ 40% = profitable อย่างยั่งยืน

### ข้อควรระวัง — อย่ากำหนด RR "ลอยๆ"

```
⚠️ RR ที่ดีต้องมาจาก market structure
    — ไม่ใช่ตั้ง RR สูงลิ่วแล้ว TP ไม่ถึง

✅ TP = OB/FVG ถัดไป (มีเหตุผล — ระดับที่ตลาดเคยตอบสนอง)
❌ TP = 100 pip (ไม่มีเหตุผล — แค่หวังว่ามันจะไปถึง)
```

**วิธีตั้ง RR ที่สมเหตุสมผล:**

```
1. เปิดกราฟ → หา OB/FVG ฝั่งตรงข้าม
2. วัดระยะ SL → TP
3. คำนวณ RR
4. ถ้า RR < 1:2 → ข้าม trade (หรือรอราคาใกล้กว่า)
5. ถ้า RR ≥ 1:2 → OK — ได้ trade ที่มีเหตุผล
```

> **RR 1:2 ที่มาจาก OB/FVG = 100× ดีกว่า RR 1:5 ที่เดาสุ่ม**

## 🔹 การจัดการความเสี่ยงขั้นสูง

### 1. Trailing Stop — เลื่อน SL ตามราคา

```
เข้า Buy ที่ 4500 SL 4497
ราคาไป 4510 → เลื่อน SL ไป 4500 (breakeven)
ราคาไป 4520 → เลื่อน SL ไป 4505 (+5 pip)
ราคาไป 4530 → เลื่อน SL ไป 4515 (+15 pip)

→ ถ้าราคากลับ: ได้กำไรแล้ว VS ตอนแรกอาจเสีย
```

### 2. Scaling — เพิ่ม/ลด Lot ตาม Equity

```
Equity Curve ขึ้น → lot size เพิ่มตาม (แต่ไม่เกิน guideline)
Equity Curve ลง → lot size ลด (ปกป้องทุน)
```

### 3. Correlation Risk

```
อย่าเทรดคู่ที่ correlated พร้อมกัน
เช่น XAUUSD (ทอง) กับ USDJPY (เงินเยน) — มี correlation

เปิด Buy XAUUSD + Buy USDJPY พร้อมกัน
→ USD แข็ง → XAUUSD ลง, USDJPY ขึ้น — เหมือนเดิม
แต่ถ้าอ่อน → เสียทั้งคู่!

→ ควรรู้ correlation ก่อนเปิดหลาย trade
```
